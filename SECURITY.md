# Security Policy

CognitiveX builds a memory layer that holds people's personal context. We take
the security and privacy of that data seriously.

## Reporting a vulnerability

**Please do not open a public issue for security problems.**

Email **security@cognitivx.io** with:

- a description of the issue and its impact,
- steps to reproduce (a proof of concept if you have one),
- any affected versions, endpoints, or components.

We aim to acknowledge reports within **2 business days** and to provide a
remediation timeline after triage. We'll keep you updated as we work toward a
fix, and we're happy to credit you once the issue is resolved (or to keep your
report confidential if you prefer).

## Scope

In scope: the iCog service (`icog.app`, `*.cognitivx.io`), the `cogix` SDK, and
the `cogx` CLI.

Out of scope: findings that require physical access, social engineering, or
denial-of-service against our infrastructure.

## Supported versions

We support the latest released version of each component. Please confirm an
issue reproduces on the current release before reporting.
